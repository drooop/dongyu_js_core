Status: PASS.

Findings:

- Current implementation and docs still use `UIPUT/ws/dam/pic/de/sw/...`.
- `packages/ui-model-demo-server/server.mjs` derives endpoint topics from Model 0 `mqtt_topic_base`; current default fallback is `UIPUT/ws/dam/pic/de/sw`.
- `packages/worker-base/src/runtime.mjs` and `scripts/worker_engine_v0.mjs` validate the old 9-segment endpoint topic shape.
- `deploy/sys-v1ns/mbr/patches/mbr_role_v0.json`, `deploy/sys-v1ns/remote-worker/patches/*.json`, and `deploy/sys-v1ns/workspace-manager/patches/*.json` contain old `/sw/` topic bases.
- 0375-era docs/tests explicitly state that request and response use the same endpoint topic.

Decision:

- The two reported issues are real and should be fixed as a hard-cut contract change.
- New current topic form is `UIPUT/<ws>/<dam>/<pic>/<de>/<worker_id>/<model_id>/<pin>`.
- New response rule is: request packet publishes to `topic`, carries `response_topic`, and response packet publishes to `response_topic`.
- `response_topic` is generated by UI Server from `reply_target_*`: `<mqtt_topic_base>/<reply_target_worker_id>/<reply_target_model_id>/<reply_target_pin>`.
