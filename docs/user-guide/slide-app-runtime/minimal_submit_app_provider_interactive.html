Input + Submit Button + Display Label

一个第三方滑动 APP 提供方需要理解的最小闭环：填表定义 UI，按钮提交临时 ModelTable records，submit 程序模型写回显示 label。

提供方只需要交付一个最小闭环

你不需要手写宿主路由，也不需要知道安装后的正式 model id。先把下面这个最小 APP 用 ModelTable records 表达出来。

单元格地图：六个 cell 构成最小 APP

完整示例仍在 `minimal_submit_app_provider_guide.md` 中，这里只解释每个 cell 的职责。

(0,0,0)Root + program
放 `model_type`、`app_name`、`host_ingress_v1`、`submit_request`、`handle_submit`、`display_text`。
(2,0,0)Container
页面根容器，`ui_layout=column`，让子组件从上到下排列。
(2,1,0)Title Text
页面标题，`ui_text=Minimal Submit App`。
(2,2,0)Input
读写 `input_text`，`commit_policy=on_submit`，避免输入时每个字符都正式提交。
(2,3,0)Submit Button
提交 `text` record 到 `submit_request`。
(2,4,0)Display Text
读取 `display_text`，展示程序模型写回的结果。

按钮 payload：必须是临时 ModelTable records

按钮不发普通对象，也不直接改 `display_text`。它只提交下面这种 records。

[
  { "id": 0, "p": 0, "r": 0, "c": 0, "k": "__mt_payload_kind", "t": "str", "v": "ui_event.v1" },
  { "id": 0, "p": 0, "r": 0, "c": 0, "k": "text", "t": "str", "v": "hello provider" }
]
安装后宿主会 remap `model_id: 0`。提供方交付包里仍然只写临时 `id: 0`。

submit 程序模型：读取 text，写回 display_text

这是本例的完整程序逻辑。正式显示结果必须由它写回，而不是由按钮直接写。

const records = Array.isArray(label && label.v) ? label.v : [];
const readPayload = (key, fallback = '') => {
  const record = records.find((item) => (
    item
    && item.id === 0
    && item.p === 0
    && item.r === 0
    && item.c === 0
    && item.k === key
  ));
  return record && Object.prototype.hasOwnProperty.call(record, 'v') ? record.v : fallback;
};

const text = String(readPayload('text', '')).trim();
const displayText = text ? `Submitted: ${text}` : 'Submitted: (empty)';
V1N.addLabel('display_text', 'str', displayText);
V1N.addLabel('last_submit_payload', 'json', records);
return;
不要使用 `ctx.writeLabel` 或 `ctx.getLabel` 作为新示例写法。

打包自检：zip 里只需要一个 JSON

交付包结构保持极简，宿主导入器负责 materialize、mount 和自动补宿主 adapter。

`minimal-submit-app.zip` 只包含 `app_payload.json`。
`app_payload.json` 是 records array，不包含 `op` 或安装后的 `model_id`。
root 有 `host_ingress_v1`，指向 `submit_request`。
按钮 `value_ref` 中的 `text` 来自 `input_text`。
`handle_submit` 只写当前模型内 label。
显示节点读取 `display_text`，不保存自己的业务 truth。
