最小 Submit 双总线示例

这份交互式文档说明 provider 如何准备一个只含 app_payload.json 的 Zip，UI Server 如何安装它，以及 Submit 如何经 message_role / endpoint_worker_id / origin_worker_id / reply_target_worker_id records 完成真实双总线回流。

topic: UIPUT/ws/dam/pic/de/sw/R1/3000/submit1visible result: Submitted: <输入内容>

真实链路

UI click -> Model 0 -> Matrix -> MBR -> remote provider public pin -> same endpoint topic response -> reply_target records -> ui-server -> local UI model
ui_bind_json writes value_ref to click_chain
click_chain -> root_routes -> submit_request -> submit_request_wiring -> handle_submit:in
handle_submit writes business records to submit1 pin.out
generated host egress adapter writes mt_bus_send_in and pin.bus.mb.out
MBR publishes endpoint topic
UIPUT/ws/dam/pic/de/sw/R1/3000/submit1
R1 returns pin_payload.v1 on the same endpoint topic with message_role=response
UI Server materializes display_text / remote_status / last_submit_payload / submit_inflight

topic 只表达 endpoint

R1 / 3000 / submit1 是远端 worker、远端 model、远端公开 pin。请求和回包都用这个 topic；UI Server 本地 model 2000 不在 topic 中。

方向和回包目标在 payload records

message_role=request 表示触发远端程序；message_role=response 表示这是回包，远端 runtime 会忽略它，避免重复触发。

reply_target_worker_id、reply_target_model_id、reply_target_pin 指向 ui-server-U1 / 2000 / result。

完整 patch label 说明

test_files/minimal_submit_dual_bus_app_payload.json 是一个 61 条 record 的 ModelTable records array；test_files/minimal_submit_dual_bus.zip 内只有 app_payload.json。

类别labels作用
metadatamodel_type, app_name, source_worker, slide_capable, slide_surface_type, from_user, to_user, ui_authoring_version, ui_root_node_id让 Workspace 识别并显示这个滑动 App。
stateinput_text, display_text, remote_status, submit_inflight, last_submit_payload页面状态与回包结果。
bushost_ingress_v1, remote_bus_endpoint_v1, dual_bus_model, submit_request, submit1, submit_request_wiring, root_routes, handle_submit声明 ingress、egress 与程序链路。
UIui_node_id, ui_component, ui_layout, ui_gap, ui_parent, ui_order, ui_title, ui_placeholder, ui_bind_json, ui_label, ui_variant把模型表渲染成页面。
bindingsclick_chain, ui_text_ref_model_id, ui_text_ref_p, ui_text_ref_r, ui_text_ref_c, ui_text_ref_k按钮事件与显示文字读取。

Submit 类提交按钮

按钮 Cell labels 必须包含 ui_component=Button、ui_label=Submit、click_chain pin.out、ui_bind_json。其中 ui_bind_json.write.pin 指向 click_chain，value_t 是 modeltable，value_ref 准备 __mt_payload_kind=ui_event.v1、text、source。

{
  "write": {
    "pin": "click_chain",
    "value_t": "modeltable",
    "value_ref": [
      { "k": "__mt_payload_kind", "t": "str", "v": "ui_event.v1" },
      { "k": "text", "t": "str", "v": { "$label": { "k": "input_text" } } },
      { "k": "source", "t": "str", "v": "ui_button" }
    ]
  }
}

Root 入口

click_chain 经 root_routes 进入 submit_request，再经 submit_request_wiring 进入 handle_submit:in。

业务出口

handle_submit 更新 input_text、submit_inflight、last_submit_payload、remote_status，并把业务 records 写到 submit1 pin.out。dual_bus_model.egress_pins 必须包含 submit1，remote_bus_endpoint_v1 必须声明 R1 / 3000。

生效顺序

ui_bind_json -> click_chain -> submit_request -> handle_submit:in -> submit1 pin.out -> generated host egress adapter。

多个提交按钮

多个提交按钮应拆开：approve_click_chain -> approve_request -> handle_approve -> approve1。不要复用同一个 root pin，除非它们确实是同一类业务。

UI Server 安装过程

  1. 读取 Zip，只接受 app_payload.json。
  2. 拒绝 op、正式 model_id、pin.bus.*、ui.egress.binding.v1、pin.connect.model、route.reply_to in zip、reply_target_*。
  3. 分配 installed model id，例如 2000。
  4. 写 Workspace 侧边栏条目，并在 Model 0 的 Workspace mount cell 写 model.submt。
  5. 写 deletable、installed_at、imported_bundle_model_ids、import_root_temp_id。
  6. 生成 host_ingress_generated_model0_labels、host_ingress_generated_mount、host_ingress_generated_root_labels。
  7. 生成 host_egress_generated_model0_labels、host_egress_generated_mount、ui_egress_submit1_binding / ui.egress.binding.v1、imported_host_submit_、imported_submit1_、bridge_imported_submit1_to_mt_bus_send_。

安装后：imported root submit1 先到 Model 0 mount cell，再到 Model 0 (0,0,0) 的 mt_bus_send_in，最后写入 pin.bus.mb.out。

Remote worker R1 的 Model 3000

R1 需要填 submit1_route：root submit1 -> (1,1,1).submit1_in -> submit1:in，再由 submit1:out -> (1,1,1).submit1_out -> root result。

root `submit1` -> `(1,1,1).submit1_in` -> `submit1:in`

程序只处理 message_role=request 的 payload。它读取 payload record 内的 text，返回 message_role=response 的 pin_payload.v1。返回 records 的 endpoint_* 仍保持 R1 / 3000 / submit1，reply_target_* 才指向 UI Server 本地模型。

Zip 导出

导出接口：/api/slide-apps/<modelId>/export.zip。导出的 Zip 只包含 provider-authored app_payload.json，不包含安装态 labels。

禁止残留：不要在 ZIP 中提供 route.reply_to、source_model_id、旧 worker/R1/model/3000/pin/submit1 topic、pin.connect.model 或 raw resultPayload。
