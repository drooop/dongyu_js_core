ModelTable-driven Slide App

滑动 APP 从填表到外发的实际链路

这张可视化图只讲 current truth：UI 是投影，正式事件进入 Model 0，运行时数据用临时 ModelTable record array，外发统一经 Model 0 管理总线。

1. 编写：把 APP 写成 ModelTable records

滑动 APP 不是一段 HTML 字符串，而是一组能被 materialize 的临时 ModelTable records。

root metadata

model_type、app_name、slide_capable、ui_root_node_id 让 Workspace 能识别和打开 APP。

UI 投影层

cellwise.ui.v1 用多个 cell 表示 Container、Text、Input、Button、布局和父子关系。

程序层

pin.in、pin.connect.label、func.js 声明后端处理链。

外发层

需要发消息时，APP 写自己的 root pin.out，不要直接发 Matrix / MBR / MQTT。

2. 安装：从 zip 到 Workspace mount

安装交付只负责把 APP materialize 到 Workspace，不处理运行时业务。

zip包含 app_payload.json
/api/media/upload上传为 media object
mxc://...得到可引用 URI
importer truth记录 URI、状态和结果
importer click pin触发导入程序
materialize / mount分配正式 modelId，并在 Model 0 写 model.submt

3. 自动建引脚：宿主只接公开边界

APP 声明 host_ingress_v1 后，安装器会生成宿主 adapter，不会把内部 helper 直接暴露给外部。

Model 0 rootimported_host_submit_<modelId> as pin.bus.in
Model 0 routepin.connect.model routes to imported root relay
Imported root relay__host_ingress_submit as pin.in
Imported boundary pinrelay to declared submit_request
如果 APP 还声明外发能力，安装器会在 Model 0 mount cell 和 Model 0 root 上补 host egress relay、bridge function 和 pin.bus.out。

4. 点击到后端：正式事件先进 Model 0

按钮点击不直接改目标业务 truth。正式提交必须是 bus_event_v2，且 value 已经是临时 ModelTable record array。

Button clickrenderer reads ui_bind_json.write
bus_event_v2bus_in_key + temporary ModelTable payload
POST /bus_eventremote store submits to server
Model 0 pin.bus.inserver writes k=<bus_in_key> t=pin.bus.in
pin routepin.connect.model reaches target model
target cell pin.inmt_bus_receive dispatches to target pin

5. 程序触发：模型表运行处执行

后端程序由目标 cell 的 pin 收到值后触发。函数读取的是 label.v，并通过 V1N 或 root 默认链写回模型表。

submit_request

业务入口 pin.in。

submit_request_wiring

把入口接到 handle_submit。

handle_submit

后端 func.js 处理临时 ModelTable payload。

submit

需要外发时写 root pin.out。

不要在新函数里使用旧 ctx.writeLabel / ctx.getLabel 口径。

6. 外发回流：统一走 Model 0 管理总线

APP 只表达“我要外发什么”；真正 transport 由 Model 0 边界统一处理。

app root pin.out例如 root submit
host / mount relay安装时自动生成
Model 0 bridge function写 mt_bus_send_in
Model 0 mt_bus_send构造 bus_send.v1
pin.bus.outMatrix / MBR / MQTT 统一消费
owner materialization回包经 Model 0 写回目标模型
